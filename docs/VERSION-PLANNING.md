# MemoirOS 版本规划

## 版本策略

MemoirOS 采用语义化版本控制 (Semantic Versioning)：

```
主版本号.次版本号.修订号 (MAJOR.MINOR.PATCH)
```

- **MAJOR**: 不兼容的 API 变更
- **MINOR**: 向后兼容的功能新增
- **PATCH**: 向后兼容的 Bug 修复

## 版本命名规范

### Pre-release 版本

- `alpha`: 内部测试版本，功能不完整
- `beta`: 公开测试版本，功能基本完整
- `rc`: Release Candidate，候选发布版本

示例：
- `0.2.0-alpha.1`
- `0.2.0-beta.1`
- `1.0.0-rc.1`

### Build 元数据

用于标识特定构建：

- `0.2.0+20260330`
- `1.0.0+build.123`

## 兼容性承诺

### v1.x 系列

- 保证向后兼容的 API 变更
- 弃用功能至少保留 2 个次版本
- 重大变更提前 2 个版本通知

### v2.0 及以后

- 遵循语义化版本控制
- 重大变更提供迁移指南
